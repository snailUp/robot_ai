 **提交格式**：
 ```
 <类型>: <简短描述>

 ```
 **类型说明**：
 | 类型     | 用途         | 示例                       |
 |---------|-------------|----------------------------|
 | feat    | 新功能       | feat: Add new map   |
 | fix     | Bug 修复     | fix: Fix map loading issue |
 | refactor| 重构         | refactor: Refactor map loading logic |
 | docs    | 文档         | docs: Update map loading documentation |
 | style   | 格式         | style: Format code         |
 | perf    | 性能         | perf: Optimize map loading performance |
 | test    | 测试         | test: Add test cases       |
 | chore   | 构建/工具     | chore: Update resource folder structure |

 **提交规范**：
 - 提交信息应使用英语，简洁明了
 - 简短描述应不超过50个字符，概括提交的主要内容
 - 详细描述应详细说明提交的原因、内容和影响
 - 提交应遵循单一职责原则，每个提交只包含一个逻辑相关的更改
 - 提交前应确保代码通过测试和代码审查
